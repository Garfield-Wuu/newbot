# `asr.cfg` 与 `/asr_id`（`std_msgs/Int32`）对照

## 说明

- **`wakeup_process`** 订阅 **`/asr_id`**，在 [`asr.cpp`](../src/asr.cpp) 中用 **`hex2dec(asr_id)`** 得到 **1-based 行号**，再减 1 作为 `asr.cfg` 解析结果数组下标。
- **`hex2dec` 规则**：把 `asr_id` 当作单字节，格式化为两位小写十六进制字符串 `"%02x"`，再把这两个字符**当作十进制数字**解读：  
  `行号 = (c0 - '0') * 10 + (c1 - '0')`。  
  例如字节 **`0x10`** → 字符串 **`"10"`** → 行号 **10**；字节 **`0x01`** → **`"01"`** → 行号 **1**。
- 因此发布 **`Int32` 的 `data` 不是简单的「配置文件行号」**。下表给出与 **每一行配置** 对应、且十六进制两位均为 **0–9** 的**推荐**整型值；同一行也可能对应其它字节值，此处不展开。
- **口令**：对应 `asr.cfg` 里 **`=` 与 `@` 之间**一段；原文用 **`|`** 分隔多种等价说法，下表中改为 **顿号** 列出，便于阅读。

## `rostopic pub` 格式（自行替换 `data`）

```bash
rostopic pub -1 /asr_id std_msgs/Int32 "data: <推荐 data>"
```

需已 **`source ~/newbot_ws/devel/setup.bash`**，且 **`wakeup_process`** 在跑、`/enable_wakeup` 为 true 时才会走完整 TTS/动作链路。

## 与 C++ 数组下标

- `hex2dec(data)` 得到 **行号 1…99**（在合法映射范围内）。
- `asr.cfg` 中第 **L** 行对应 **`id_index = L - 1`**（`get_asr_reply` 内）。  
  若 `hex2dec` 超出已解析行数，会返回错误提示字符串。

## 维护

若增删 `asr.cfg` 行，请按 [`asr.cpp`](../src/asr.cpp) 中 `hex2dec` 重新核对本表，或运行工作区内用于生成 `usb_voice_cmd` 的同一套逻辑脚本重新制表。
